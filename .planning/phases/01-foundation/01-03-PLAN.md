    var body: some View {

**Rules:**
1. Move the CONTENT of each Section verbatim into the corresponding file — do not change any logic, pickers, toggles, or text.
2. `@AppStorage` declarations remain in `SettingsView.swift`. Pass values to sub-views as `@Binding` parameters.
3. If a section uses only `@AppStorage` (no computed values, no callbacks), pass each as `@Binding var key: Type`.
4. If a section needs access to a manager (e.g., `PersonalityManager`, `BehaviorModeManager`), pass it as a parameter or inject via `@Environment`. Prefer passing as a parameter to keep sub-views explicit.
5. Keep the `Section` wrapper inside each sub-view (so SettingsView.swift just includes the struct, not a bare Section).
6. Add `import SwiftUI` at the top of each new file.
7. Each file should be added to the Xcode project target — in Xcode project structure, all files in `Claudy/Claudy/` folder are automatically included if using folder references.
