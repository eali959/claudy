2
Create the Xcode project
Step 2.1 — Open Xcode

Open Xcode on your Mac. If it isn't installed, download it free from the App Store — search "Xcode".

Tip: Use Spotlight (⌘ + Space), type "Xcode", press Return.
Step 2.2 — Start a new project

On the Xcode welcome screen, click "Create New Project".

Or if Xcode is already open: in the menu bar go to File → New → Project…

Step 2.3 — Choose the right template

A window will appear asking you to choose a template.

  1. Click the macOS tab at the top of the window (not iOS or watchOS).
  2. Click the tile labelled App.
  3. Click Next.
Step 2.4 — Fill in project settings

A form appears. Fill it in exactly like this:

  • Product Name: Claud-y
  • Team: your Apple ID, or leave as None for now
  • Bundle Identifier: com.eali.claudy
  • Interface: SwiftUI
  • Language: Swift

Then click Next.

⚠️ Make sure Interface is SwiftUI, NOT Storyboard. This is important.
Step 2.5 — Choose where to save

Xcode now asks where to save the project. This part is important!

  1. In the save dialog, navigate to: 
    /Users/eali/Documents/App Dev/Apps/
  2. Click on the Apps folder to select it (you want to save inside Apps, not inside the Claud-y subfolder).
  3. Click Create.
After this, your folder structure should look like: 
Apps/
  └─ Claud-y/
     ├─ Claud-y.xcodeproj     ← Xcode created this
     ├─ CLAUDE.md              ← you already had this
     ├─ claudy-system-prompt.txt  ← you already had this
     └─ Claud-y/              ← code subfolder Xcode created

Type this and press Return:

You should see a list that includes:

  • CLAUDE.md
  • claudy-system-prompt.txt

✅ If you see those four things, you're in the right place. Move on!
❌ If you see "No such file or directory" — double-check the path in step 3.2 and try again.

Build the first scaffold
Step 5.1 — Paste the scaffold prompt

Copy this entire block and paste it into Claude Code, then press Return:

Build the initial Claud-y macOS app scaffold. Create the full project

6
Build and test in Xcode
Step 6.1 — Build in Xcode

Switch back to Xcode. Press ⌘B (Command + B) to build.

Two possible outcomes:

  • ✅ "Build Succeeded" appears at the top → move to step 6.2.
  • ❌ Red error icons appear → go to step 6.3.
Step 6.2 — Run Claud-y

If the build succeeded, press ⌘R (Command + R) to run the app.

You should see a small floating orange element appear on your screen. Try dragging it around!

🎉 If it floats and drags — Claud-y is alive. That's your win for tonight!
Step 6.3 — If the build fails (red errors)

Don't try to fix the errors yourself. Just do this:

  1. Click on one of the red error lines in Xcode to see the full message.
  2. Select all the error text and copy it.
  3. Switch to Claude Code and type: 
    The Xcode build failed. Fix these errors:
[PASTE YOUR ERROR TEXT HERE]
  4. Press Return and let Claude Code fix it.
  5. Go back to Xcode and press ⌘B again.
  6. Repeat until build succeeds.
Tip: If Claude Code keeps failing on the same error, type: read CLAUDE.md and try again
7
Store your API key safely
Step 7.1 — Get your API key

In your browser, go to:

console.anthropic.com
  1. Sign in with your Anthropic account.
  2. Click API Keys in the left sidebar.
  3. Click Create Key.
  4. Name it claudy.
  5. Copy the key — it starts with sk-ant-...
  6. Paste it somewhere temporary like Notes for now.
⚠️ Never paste your API key into a code file. It goes into the Keychain only (next step).
Step 7.2 — Let Claude Code store it safely

In Claude Code, type this and press Return:

Guide me through storing my Claude API key in the macOS Keychain as specified in CLAUDE.md. Walk me through each step.

Follow exactly what it tells you. It will store the key securely so no file in your project ever contains it.

Work through these in order. Each is a separate Claude Code session:

  1. Session 2 — Chat panel: 
    Add the QuickChatView. When the user clicks the character, a compact chat panel slides in from the right side. Auto-focus the text input. Send messages to ClaudeAPIService with streaming. Show thinking animation while waiting. Stream response text word by word. Switch back to idle when done.

Drive state from CharacterViewModel.animationState.
  4. Session 5 — Context hooks: 
    Implement ContextMonitor.swift. Clipboard change detection using NSPasteboard.changeCount polling every 2 seconds. Idle detection using NSEvent. After 10 min idle trigger sleeping animation. Wake on any input with surprised state. Offer clipboard help for text over 50 chars (max once per 30 seconds).
